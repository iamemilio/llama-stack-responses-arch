HTTP Request
Client sends POST /v1/responses with model, input, tools, and optional stream: true.
fastapi_routes.py
Route Handler
Parses request, calls impl.create_openai_response(). If result is an async iterator, wraps in SSE StreamingResponse.
fastapi_routes.py → api.py
Validation & Preparation
Resolves previous_response_id / conversation chains. Applies prompt templates. Checks guardrails config.
openai_responses.py
Create Streaming Pipeline
Both streaming and non-streaming go through the same internal stream. Non-streaming consumes it and returns the final object.
openai_responses.py → streaming.py
Agent Loop
Core execution cycle inside StreamingResponseOrchestrator:
1. Guardrails
Run safety shields on input. Violations produce a refusal and exit.
2. Inference
Call Inference.openai_chat_completion(stream=True). Translate chat completion chunks into Responses stream events.
3. Tool Execution
If model returns tool calls, ToolExecutor dispatches them (web_search, file_search, MCP, user functions). Results appended to messages.
↺ Repeat steps 1–3 until no more tool calls
streaming.py, tool_executor.py
Persist Response
ResponsesStore upserts incrementally during streaming so in-progress responses are retrievable via GET.
responses_store.py
Terminal Event
Emits response.completed, response.incomplete, or response.failed. Stream ends.
